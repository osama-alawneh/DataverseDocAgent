## Successes

1. **Permission checker works against a real Dataverse environment.** AC6 of story 2.3 was executed live against `orgd76c9cf3` on 2026-04-17. Four real issues surfaced and were fixed before any customer exposure.
2. **PRD errors caught before shipping.** The PRD privilege list carried a phantom `prvReadRolePrivilege` and the wrong `Read SavedQuery` naming. Both corrected; count 13 → 12.
3. **SharePoint DMI whitelist.** `KnownHarmlessExtraPrivileges` prevents every SharePoint-enabled tenant from seeing noise in `extra[]`. Discovery was only possible through live execution.
4. **Plain-English trust docs.** `docs/setup-guide.md` and `docs/privacy-policy.md` are specific and defensible — no legal boilerplate, explicit NFR mapping (NFR-007, NFR-010, NFR-012, NFR-013).
5. **Least-privilege role shipped.** `DataverseDocAgent Reader` role at 12 Organisation-scoped privileges, committed as `DataverseDocAgentSecurityRole_1_0_0_6_managed.zip`.

## Challenges

1. **Story 2.3 "validated-by-design" near-miss.** The initial `done` transition relied on XML inspection against `SecurityCheckService.RequiredPrivileges`. Live AC6 surfaced four distinct bugs that unit tests could not detect:
   - `GetApplicationUserIdAsync` required `prvReadUser` (not in role) — fixed via `WhoAmIRequest`.
   - `prvReadRolePrivilege` does not exist in Dataverse — removed from PRD, service, tests, README.
   - Entity `savedquery` uses privilege name `prvReadQuery` (no "Saved" prefix) — required-list entry corrected.
   - SharePoint DMI auto-grants 4 baseline privileges and reinserts them if stripped — whitelisted.
2. **Three sources of truth for the privilege list.** PRD §5.4, README, `customizations.xml`, plus the code constant in `SecurityCheckService`. The 13 → 12 correction required changes across 6 files in lockstep.
3. **High review-patch volume.** Review fires after developer says `done`. Pattern visible in every story record.
4. **Deferred-item accumulation.** Five Epic 2 items carry forward: rate limiting, TenantId wiring, middleware body-logging audit, GDPR legal review, CI gap.

## Key Lessons

1. **Live validation is not optional for external-system ACs.** "Validated by design" is not validated.
2. **PRD errors propagate.** Single-source-of-truth discipline matters; implementation code should be canonical for protocol-level facts.
3. **Public doc claims bind future work.** Privacy policy retention claim commits us to shipping story 3.2 `IDocumentStore` before any public launch.
4. **Deferred is not free.** Each deferral shapes the next epic's priorities.
