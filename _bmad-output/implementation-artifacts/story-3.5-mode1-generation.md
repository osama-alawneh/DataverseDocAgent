# Story 3.5: Mode 1 — POST /api/document/generate Endpoint and Full Request-to-Download Flow

Status: done

## Story

As a D365 consultant,
I want to call POST /api/document/generate with my credentials and receive a .docx via the async job flow,
so that the complete request-to-download journey works end-to-end: submit → poll → download.

## Acceptance Criteria

1. `POST /api/document/generate` accepts a JSON body with: `environmentUrl` (string, required), `tenantId` (string, required), `clientId` (string, required), `clientSecret` (string, required). The endpoint validates required field presence and returns a structured error with `code: "INVALID_REQUEST"` for missing fields. On valid input, it returns HTTP 202 Accepted with `{ jobId: "<uuid>" }` within 2 seconds of receipt.
2. Credentials submitted in the request body are never logged at any point in the request or background processing lifecycle (FR-034, NFR-007).
3. The background pipeline performs a lightweight Dataverse connection test as the first step. If the connection test fails, the job is marked `Failed` with `code: "CREDENTIAL_REJECTED"` and credentials are discarded immediately.
4. On successful connection, the `AgentOrchestrator` is invoked with the three tools from Story 3.4 (`list_custom_tables`, `get_table_fields`, `get_relationships`) registered. Claude is prompted to gather environment metadata and produce a structured JSON output with four sections: `executive_summary`, `tables`, `fields`, `relationships`.
5. The `executive_summary` section of Claude's output includes: `environmentName` (from `get_organisation_metadata`), `scanDate` (ISO 8601 UTC), `complexityRating` (string: "Low" | "Medium" | "High"), `tableCount` (int), `fieldCount` (int), `relationshipCount` (int). The `complexityRating` is computed deterministically in C# from counts — not generated by Claude. Scoring: Low = <10 tables AND <100 fields; Medium = 10–50 tables OR 100–500 fields; High = >50 tables OR >500 fields.
6. `DocxBuilder.Build(structuredOutput)` produces a valid `.docx` file containing sections 1–4 as defined in PRD Section 8.1: (1) Executive Summary, (2) Custom Tables, (3) Field Catalogue, (4) Relationship Map. The document opens in Microsoft Word 2016 or later without errors.
7. The generated `.docx` is stored via `IDocumentStore.StoreAsync(bytes, ttl: 24h)`, the job is marked `Ready` with the returned download token, and `GET /api/jobs/{jobId}` returns `{ status: "ready", downloadToken: "<token>" }`.
8. `GET /api/download/{token}` returns the `.docx` file with correct Content-Type and Content-Disposition headers.
9. All structured failure modes return the correct job state and error code: credential rejection → `CREDENTIAL_REJECTED`; Dataverse connectivity error → `DATAVERSE_ERROR`; Claude API error → `AI_ERROR`; generation timeout (10 minutes) → `GENERATION_TIMEOUT`. In all cases, credentials are discarded.
10. Elapsed time from `POST /api/document/generate` receipt to job status "ready" is captured by the background service and logged (timing only — no credential data) for NFR-001 baseline comparison.

## Dev Notes

- **Claude output format:** The prompt must specify the exact JSON structure Claude should return. Be prescriptive: "Return a single JSON object with these exact keys: ...". If Claude wraps the JSON in markdown code fences (`\`\`\`json`), strip them in `DocumentGenerateService` before `JsonSerializer.Deserialize()`.
- **`AgentOrchestrator` with Mode 1 tools:** The orchestrator from Story 1.3 was built for POC. For production, it needs to handle the case where Claude calls `get_table_fields` once per table (potentially 50+ calls in a large environment). The 10-iteration guard in the POC must be raised or replaced with a termination condition based on `stop_reason: end_turn`.
- **DocxBuilder with `DocumentFormat.OpenXml`:** Use `WordprocessingDocument.Create(memoryStream, WordprocessingDocumentType.Document)`. Build the body with `Paragraph` and `Table` elements. For the Mode 1 MVP, simple formatting is acceptable — bold headings, plain body text, simple tables. Full styling can be a Phase 3 improvement.
- **`MemoryStream` for .docx:** `DocxBuilder.Build()` should return `memoryStream.ToArray()` after saving and closing the `WordprocessingDocument`. Do not return the stream itself — it may be closed by the time the caller reads it.
- **Credential disposal on failure:** In the catch blocks, `task.Credentials` goes out of scope when `RunAsync` returns. There is no explicit `Dispose()` needed — `EnvironmentCredentials` holds only managed strings. The GC handles collection. What matters is that no reference is retained in a static field, cache, or log.
- **`get_table_fields` called per table:** For an environment with 30 custom tables, this means 30 tool calls. Each is a separate API call to Dataverse. This is the expected pattern — it's why the 5-minute target is TBD from POC. Monitor elapsed time per tool call and log totals.
- **`keyObservations` from Claude:** This is the only section of the executive summary that is AI-generated (FR-011 requires "key observations are AI-generated and specific to the scanned environment"). All other executive summary fields are deterministic. Claude generates 3–5 plain-English observations based on what it found during tool calls.
